# Moment Music — Overview

## Research Origins

Moment Music grew out of a research project on **Controllable AI Sonification via LLM** — a system that autonomously transforms data into sonified outputs using AI to interpret, control, and generate sound.

The original system architecture defines four stages:

1. **Data Provisioning** — Sourcing data from diverse inputs: web crawlers, sensors (environmental, physiological), and creative sources (e.g. "Listen to Wikipedia" turning edits into sounds)
2. **Data Interpretation** — An LLM replaces manual interpretation, translating raw data into prompts that guide sound generation. Critically, the LLM also acts as a **dynamic controller** — selecting and configuring downstream AI models based on the interpreted context
3. **Data Generation** — AI models convert text to sound/music (audio and MIDI formats)
4. **User Interface** — Originally Gradio for rapid prototyping; now evolved into a full React-based creative workspace

This research foundation means the LLM isn't just a prompt generator — it's the intelligent orchestrator of the entire pipeline, deciding *what* to generate and *how* to configure the generation models.

## Vision

Every moment has a sound. Moment Music is an AI-powered creative platform that translates lived experiences into personalized music. By capturing the texture of a moment — through words, sounds, movement, or environment — users create unique soundscapes that reflect how a moment truly feels.

## Core Concept

The platform offers **four creation modes**, each capturing a different sensory dimension:

| Mode | Input | Captures |
|------|-------|----------|
| **Write** | Text journal, image upload/camera | Inner thoughts, visual context |
| **Listen** | Microphone ambient capture | Sonic environment |
| **Move** | Device motion sensors | Physical rhythm, energy |
| **Be** | Location → weather + time | Environmental atmosphere |

These inputs feed into an **AI interpretation pipeline** that:
1. Analyzes the multimodal context
2. Extracts emotional mood and atmosphere
3. Generates a structured music prompt (or lyrics, or narration script)
4. Produces audio through specialized AI engines
5. Generates album cover art in parallel

## Output Types

Each mode can produce three types of audio output:

- **Instrumental** — Pure music generated by ACE-STEP (default), HeartMuLa, Stable Audio Open, or Stable Audio API
- **Song** — Vocals + lyrics via ACE-STEP or HeartMuLa
- **Narration** — Spoken word via Qwen3-TTS over ambient soundtrack, mixed with FFmpeg

## Music Engines

Users choose their preferred engine per output type in the Setup page:

| Engine | Type | Best For |
|--------|------|----------|
| **ACE-STEP** (default) | Local | General-purpose, works on Apple Silicon |
| **HeartMuLa** | Local | Lyrics-conditioned music (needs 36GB+ VRAM) |
| **Stable Audio Open** | Local | Diffusion-based instrumental (needs HF_TOKEN) |
| **Stable Audio API** | Cloud | No local GPU needed (needs API key) |
| **Qwen3-TTS** | Local | Narration voice synthesis |

## User Experience Flow

```
[Entryway Portal]
    ↓ choose mode
[Write | Listen | Move | Be]
    ↓ capture moment
[Job Queue] → step-by-step progress (mood → audio → finalize)
    ↓ complete
[Moment Player] → playback, album art, download, save to library
```

### Entryway
An immersive portal screen presenting the four modes as glowing cards. Each card reveals its description on hover. The entryway sets the creative tone — this is not a utility, it's an experience.

### Creation Modes
Each mode has a dedicated capture screen with mode-specific controls. All modes share:
- Output type selector (Instrumental / Song / Narration)
- Generate button with real-time step progress
- Engine selection (via Setup page)
- Style prompt customization (via Prompts page)

### Moment Player
The output screen with:
- Album art with animated glow border
- Audio playback controls
- Track metadata (mood keywords, summary, engine, prompt)
- Download option
- Save to Library

### Library
Collection of saved soundscapes with album art thumbnails.

### Setup
Engine selection per output type + API key management (OpenAI, Stability AI, HuggingFace).

### Prompts
Style prompt customization with global defaults and per-mode overrides (Write, Listen, Move, Be).

## Current Status

All four creation modes, all three output types, and the multi-engine system are implemented and functional. The job queue handles concurrent requests with progress tracking.

## Design Philosophy

The interface is **immersive and atmospheric**, not utilitarian. Key principles:
- Dark-mode-first with glass morphism
- Animated, ambient UI that feels alive
- Minimal chrome — content and creation take center stage
- The UI itself should feel like part of the creative experience
